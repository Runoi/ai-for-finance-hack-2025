# AI for Finance Hack 2025 - RAG Агент-Аналитик

Этот репозиторий содержит решение для хакатона "AI for Finance Hack 2025". Мы строим продвинутую RAG-систему, основанную на агентской архитектуре, для ответов на сложные финансовые вопросы.

## 🚀 Архитектура

Наше решение основано на нескольких ключевых статьях и инженерных практиках:

1.  **"Агентский Цикл" (FAIR-RAG):** Вместо простого "поиск -> ответ", наша система работает в итеративном цикле. Она анализирует найденную информацию, определяет "пробелы" в знаниях и генерирует новые, уточняющие запросы, пока не будет уверена в полноте ответа.
2.  **Гибридный Поиск (Ensemble Retriever):** Мы используем 4-головый поисковый движок для максимальной полноты и точности:
    *   **Семантический поиск (FAISS):** Понимает смысл.
    *   **Поиск по ключевым словам (TF-IDF):** Находит точные термины.
    *   **Поиск по метаданным:** Быстро находит статьи по общей теме.
    *   **Графовый поиск (NetworkX):** Находит скрытые связи между документами.
3.  **Мониторинг Ресурсов:** В систему встроен `ResourceManager` для отслеживания времени, стоимости API и потребления RAM/CPU в реальном времени.

## 🛠️ Быстрый Старт

### 1. Требования

*   Python 3.12+
*   Все зависимости из `requirements.txt`

### 2. Настройка Окружения

1.  **Клонируйте репозиторий:**
    ```bash
    git clone [URL-репозитория]
    cd ai-for-finance-hack-2025
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv venv
    source ven/bin/activate  # Для Linux/macOS
    .\venv\Scripts\activate   # Для Windows
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Создайте файл `.env`:**
    *   Скопируйте файл `.env.example` (если он есть) или создайте новый файл `.env` в корне проекта.
    *   Пропишите в него ваши API-ключи, полученные от организаторов:
      ```dotenv
      LLM_API_KEY="sk-..."
      EMBEDDER_API_KEY="sk-..."
      ```

5.  **Загрузите данные:**
    *   Поместите файлы `train_data.csv` и `questions.csv` в корневую папку проекта.

### 3. Запуск в Режиме Отладки (для одного вопроса)

Это основной способ для разработки и тестирования.

1.  **Запустите скрипт:**
    ```bash
    python debug_single_question.py
    ```
2.  **Первый запуск** будет долгим (~3-5 минут), так как он создаст все поисковые индексы в папке `storage/`. **Не прерывайте его.**
3.  Последующие запуски будут быстрыми (10-15 секунд).
4.  Введите вопрос в консоли и нажмите Enter.

### 4. Запуск Полного Прогона (генерация `submission.csv`)

Этот скрипт используется для создания финального файла для сдачи.

```bash
python main.py
```
*   Он также создаст индексы при первом запуске.
*   После завершения работы в корне проекта появится файл `submission.csv`.

## ⚙️ Управление Конфигурацией

Все основные настройки и "тумблеры" для управления архитектурой находятся в файле `config.py`. Вы можете изменять его для проведения экспериментов:
*   `MAX_ITERATIONS`: Управляет глубиной агентского цикла.
*   `ENABLE_..._HEAD`: Включает/выключает "головы" ретривера.
*   `..._MODEL`: Позволяет менять модели для разных ролей (агент-аналитик, генератор).

## 🗂️ Структура Проекта

*   `main.py`: Точка входа для финального прогона.
*   `debug_single_question.py`: Наш главный инструмент для отладки.
*   `prepare_logic.py`: Логика создания всех поисковых индексов.
*   `build.py`: Скрипт для сборки проекта в один файл для сдачи.
*   `config.py`: Центральный конфигурационный файл.
*   `rag_components/`: Ядро нашего решения (пайплайн, агенты, ретриверы).
*   `utils/`: Вспомогательные модули (мониторинг, промпты).

### 🧑‍💻 **Backend & Architecture (Владелец: Я, Архитектор)**
*   **Сфера:** Основная логика в `rag_components/`, `prepare_logic.py`, `main.py`.
*   **Текущие задачи:**
    *   `[ЗАВЕРШЕНО]` Реализация базовой инфраструктуры (Спринт 1).
    *   `[В РАБОТЕ]` Реализация агентского цикла в `pipeline.py` (Спринт 2).
    *   `[ДАЛЕЕ]` Оптимизация производительности `prepare_logic.py`.
*   **Требуемые навыки:** Глубокое понимание Python, LangChain, архитектуры ПО.

### 🧠 **AI Quality & Prompts (Владелец: JS-разработчик)**
*   **Сфера:** `utils/prompt_library.py`, анализ логов, A/B-тестирование.
*   **Текущие задачи:**
    *   `[В РАБОТЕ]` Создание и итеративное улучшение библиотеки промптов для `SeaAgent`, `RefinementAgent` и `final_generator`.
    *   `[ДАЛЕЕ]` Запуск `debug_single_question.py` на 10-20 разных вопросах, анализ логов `ResourceManager` и оценка качества ответов.
    *   `[ДАЛЕЕ]` Проведение A/B-теста: сравнить `gemma-3-27b-it` и `mistral-small` в роли `GENERATOR_MODEL` по качеству и стоимости.
*   **Требуемые навыки:** Понимание работы LLM, внимание к деталям, аналитический склад ума.

### 🔍 **Data & Retrieval (Возможная роль для Участника №3)**
*   **Сфера:** `prepare_logic.py`, `rag_components/retrievers.py`, `config.py`.
*   **Возможные задачи:**
    *   **Тюнинг Графа Концептов:** Поэкспериментировать с параметрами `TfidfVectorizer` (`max_features`, `ngram_range`) в `prepare_logic.py` для извлечения более качественных концептов.
    *   **Тюнинг Весов Ретривера:** Изменить `RETRIEVER_WEIGHTS` в `config.py` и оценить, как это влияет на релевантность найденных документов на тестовых вопросах.
    *   **Исследование Реранкинга:** Изучить возможность добавления `CrossEncoder`-реранкера (например, из `sentence-transformers`) после `EnsembleRetriever` для повышения точности поиска.
*   **Требуемые навыки:** Понимание NLP (TF-IDF), работа с данными, `scikit-learn`.
